# Посібник користувача KVision

> [!ПРИМІТКА]
> Цю інструкцію з експлуатації було автоматично перекладено та відформатовано за допомогою штучного інтелекту (AI).

**KVision** — це розширена настільна програма класу VMS (Video Management System), призначена для одночасного перегляду потокового відео в реальному часі (RTSP/ONVIF) та інтеграції з рекордерами Hikvision NVR/DVR (як у режимі реального часу, так і в режимі архіву відтворення).

Програма була оптимізована для стабільності, плавної роботи (60 FPS) і мінімального споживання системних ресурсів.

## 1. Опис дій кнопок

У цьому розділі описано значення всіх графічних значків і кнопок, які використовуються в програмі.

### Верхня панель інструментів
* {ICON:quit} **Закрити вікно**: запит на підтвердження та закриття активного вікна або програми.
* {ICON:pin} **Панель закріплень**: фіксує верхню панель у розгорнутому стані або вмикає автоматичне згортання.
* {ICON:fullscreen} **Повноекранний режим**: перемикає активне вікно в повноекранний режим.
* {ICON:minimize} **Згорнути**: згортає вікно програми до системної панелі завдань.
* {ICON:options} **Параметри**: відкриває або закриває вікно параметрів конфігурації (параметри та записи).
* {ICON:new_window} **Нове вікно**: відкриває нове незалежне допоміжне вікно для потоків камери.
* {ICON:archive} **Архів**: відкриває вікно архіву записів відтворення (хронологія та календар).
* {ICON:instructions} **Інструкції**: відкриває цей посібник користувача та вікно технічної допомоги.
* {ICON:stats} **Статистика**: перемикає ковзаючу панель статистики системи (CPU, RAM, GPU, Net).
* {ICON:lock} **Блокування сітки**: вимикає налаштування поділу сітки, щоб захистити ваш активний макет.
* {ICON:hamburger} **Додаткові параметри**: відкриває ковзаючу панель інструментів для розширених розмірів поділу, співвідношень і об’єднання клітинок.

### Накладання вікна перегляду (камери)
* {ICON:snapshot} **Знімок**: робить зображення з повною роздільною здатністю без втрат і зберігає його як файл JPEG.
* {ICON:play} **Архів камери**: запускає вікно відтворення шкали часу для цієї камери (на 15 хвилин назад).
* {ICON:quick_play} **Швидке відтворення (міні-програвач)**: ініціює відтворення архіву за останні 30 хвилин у вікні перегляду, що дозволяє швидко переглядати нещодавні кадри безпосередньо на плитці камери, не виходячи з Live view.
* {ICON:grid_1x1} **Спробуйте 1:1**: відеопотік відображається в оригінальній рідній роздільній здатності без розтягування.
* {ICON:zoom_in} **Інтерактивне масштабування**: вмикає/вмикає збільшення, натискаючи та перетягуючи для вибраної області виділення.
* {ICON:zoom_out} **Скинути масштаб**: скидання цифрового збільшення та відновлення повного поля огляду камери.
* {ICON:speaker_unmute} **Вимкнути аудіо**: означає невимкнений аудіопотік; клацання вимикає потік.
* {ICON:speaker_mute} **Увімкнути аудіо**: представляє приглушений аудіопотік; клацання вмикає звук потоку.

## 2. Встановлення та запуск

### Встановлення на Arch Linux (Pacman)
Щоб встановити програму з підготовленого бінарного пакета, перейдіть в каталог `packaging/arch/` і запустіть:
```bash
sudo pacman -U kvision-2.6.2-1-x86_64.pkg.tar.zst
```
Пакет автоматично встановить програму, файл активації `.desktop` та необхідні бібліотеки Hikvision SDK до системного шляху `/usr/lib/kvision`.

### Ручна компіляція (з вихідного коду)
Якщо ви хочете скомпілювати програму вручну (наприклад, на іншому дистрибутиві Linux) замість використання готового пакету:

1. Встановіть необхідні залежності збірки та середовища виконання за допомогою менеджера пакетів. Для Arch Linux / CachyOS:
   ```bash
   sudo pacman -S base-devel cmake qt5-declarative qt5-multimedia qt5-quickcontrols qt5-quickcontrols2 qt5-svg qt5-graphicaleffects qt5-tools ffmpeg git qt5-wayland
   ```
2. Налаштуйте проект за допомогою CMake:
   ```bash
   cmake -B build -S . -DCMAKE_BUILD_TYPE=Release -DCMAKE_INSTALL_PREFIX=/usr
   ```
3. Скомпілюйте код:
   ```bash
   cmake --build build -j$(nproc)
   ```
4. Встановіть програму в систему:
   ```bash
   sudo cmake --install build
   ```

### Запуск
Програму можна запустити з системного меню або ввівши в терміналі:
```bash
kvision
```

### Усунення проблем масштабування системи (KDE Plasma)

Починаючи з версії 2.4.4, KVision автоматично виявляє та правильно застосовує часткове масштабування стільниці (наприклад, 125%, 145%, 150%), налаштоване в налаштуваннях KDE Plasma, застосовуючи власну політику проходження High DPI у Qt. Користувальницький інтерфейс тепер має правильно масштабуватися з коробки без ручного втручання.

#### Вбудована підтримка Wayland

Щоб забезпечити найкраще масштабування та продуктивність у Wayland, переконайтеся, що у вашій системі встановлено додатковий пакет `qt5-wayland`. Це дозволяє KVision працювати на Wayland замість повернення до XWayland.

Для Arch Linux / Manjaro / CachyOS:
```bash
sudo pacman -S qt5-wayland
```

Для Debian / Ubuntu / Linux Mint:
```bash
sudo apt install qtwayland5
```

#### Заміна масштабування вручну (застаріле/резервне)

Якщо ви використовуєте старішу версію, маєте незвичайну конфігурацію або просто хочете вручну встановити коефіцієнт масштабування, відмінний від решти системи, ви можете перевизначити масштабування програми за допомогою змінних середовища.

Наприклад, щоб примусово масштабувати **150%**, запустіть програму з терміналу за допомогою:
```bash
env QT_FONT_DPI=96 QT_SCALE_FACTOR=1.5 kvision
```

*(Це можна застосувати назавжди, відредагувавши засіб запуску програм `kvision.desktop` у редакторі меню KDE та додавши ці змінні до поля «Змінні середовища»).*

## 3. Керування записуючими пристроями NVR/DVR

Щоб налаштувати підключення до рекордера Hikvision:
1. Відкрийте вікно параметрів і перейдіть на вкладку **Записувачі** (значок сервера).
2. Введіть дані доступу до пристрою:
   * **IP-адреса**: мережева адреса записуючого пристрою.
   * **Порт**: мережевий порт SDK (за замовчуванням `8000`).
   * **Ім'я користувача**: ім'я користувача (наприклад, `admin`).
   * **Пароль**: Пароль для доступу до диктофона.
3. Натисніть **Підключитися та знайти** (або **Зберегти та оновити**).
4. Після успішного підключення програма автоматично виявить усі активні камери (канали), підключені до NVR/DVR, і додасть їх до списку.
5. Натискання кнопки **Створити сітку** автоматично створить макет вікна перегляду (попередньо налаштований), що містить усі активні камери з цього NVR в оптимальному макеті сітки.
6. **Відображення списків камер (NvrCamerasWindow)**: натискання кнопки піктограми монітора комп’ютера на будь-якій картці запису відкриває спеціальне вікно, у якому відображаються всі виявлені канали камер як інтерактивні плитки.
7. **Створення мініатюр (Створення мініатюр)**: у вікні списку камер NVR доступна кнопка *„Створити мініатюри”*. Якщо натиснути на нього, програмі буде наказано отримати окремі кадри з підпотоку кожного каналу у фоновому режимі, встановивши їх як мініатюри фону плитки. Це забезпечує швидкий візуальний попередній перегляд кожної камери без запуску повних прямих трансляцій.
8. **Функція «клацніть і додайте»**: програма не підтримує перетягування плиток (перетягування) із вікна списку камер до основної сітки. Призначення камери виконується простим і надійним способом: спочатку клацніть лівою кнопкою миші будь-яку плитку вікна перегляду в сітці головного екрана, щоб вибрати її (вона буде виділена яскравою рамкою), а потім натисніть зелену кнопку **"+" (Призначити активному вікну перегляду)** на плитці потрібної камери у вікні списку камер NVR. Потік миттєво завантажиться в цей слот.
9. **Статус сеансу SDK (крапковий індикатор)**: поруч із IP-адресою кожного записувача в списку є кольорова крапка стану:
   * **Зелений (УВІЙШЕНО)**: означає активний сеанс Hikvision SDK, який потрібен для керування PTZ, запитів на архів часової шкали та завантаження.
   * **Червоний (НЕ ВВІЙШУВ)**: наразі не встановлено жодного активного сеансу SDK (наприклад, перед першим запитом SDK або після ручного/автоматичної виходу). Зауважте, що живі канали камери RTSP відтворюватимуться незалежно від стану сеансу SDK.
10. **Локальне перейменування камери**: на будь-якій плитці камери в списку камер NVR натисніть піктограму **Редагувати** (олівець). Відкриється діалогове вікно, у якому можна встановити власну назву для камери. Це ім’я зберігається локально та миттєво оновлюється в живих програвачах, часовій шкалі та вікнах без зміни фізичного пристрою NVR. Ви можете будь-коли відновити ім’я за замовчуванням.
11. **Видалення запису зі списку**: поруч із кожним налаштованим записувачем у списку є червона кнопка кошика. Клацнувши її, ініціюється двоетапний протокол безпеки, призначений для запобігання випадковому видаленню:
    * **Крок 1 (Підтвердження)**: з’являється діалогове вікно під назвою *«Підтвердити видалення NVR»* із запитом, чи впевнені ви, що бажаєте видалити записувач.
    * **Крок 2 (Попередження)**: з’являється друге діалогове вікно з попередженням під назвою *«Попередження!»* із запитанням, чи ви абсолютно впевнені та усвідомлюєте, що робите.
    * **Наслідки видалення**: після прийняття другого попередження програма виходить із цього NVR у фоновому режимі, видаляє свій запис із конфігурації програми та **автоматично очищає список попередньо встановлених макетів**, видаляючи всі динамічні сітки, згенеровані для цього пристрою NVR.

## 4. Live View і Viewport Overlays

Головне вікно програми відображає живу стрічку:
* **Сітка камери**: відображає потоки або канали RTSP безпосередньо з Hikvision SDK.
* **Вибір якості потоку**: клацнувши правою кнопкою миші вікно перегляду камери, ви можете вибрати **Основний потік** для найвищої роздільної здатності або **Додатковий потік**, щоб зменшити навантаження на мережу та графічну карту.
* **Перемикання повноекранного режиму подвійним клацанням**: подвійне клацання лівою кнопкою миші на будь-якому вікні перегляду камери миттєво розгортає його, щоб заповнити всю область активного вікна (повноекранний режим з одним вікном перегляду). Повторне подвійне клацання відновлює оригінальний макет багатокамерної сітки.
* **Автоматичне приховування верхньої панелі**: верхня панель інструментів (topToolBar) може автоматично згортатися до верхнього краю екрана, коли курсор миші залишає її область (цей параметр можна налаштувати в Параметрах -> *„Автоматично згортати верхню панель”* або безпосередньо за допомогою значка шпильки на верхній панелі).
* **Кількома моніторами та допоміжними вікнами**: ви можете відкривати незалежні додаткові (допоміжні) вікна, щоб запускати різні макети сітки одночасно на кількох екранах або моніторах. Щоб відкрити нове вікно, скористайтеся комбінацією клавіш `Ctrl+N` або клацніть кнопку **"Нове вікно"** на верхній панелі інструментів. Для кожного вікна можна налаштувати власний розмір сітки та вибраний попередньо встановлений макет. Щоб забезпечити максимальну стабільність у конфігураціях із кількома моніторами, усі вікна запускаються строго по центру основного дисплея з роздільною здатністю 90% від його роздільної здатності.

### Кнопки накладання вікна перегляду
У нижньому правому куті кожної плитки/окна перегляду камери відображається панель керування з п’ятьма функціональними значками, якщо навести на неї курсор миші (залежно від налаштувань інтерфейсу користувача):
1. **Піктограма камери (миттєвий знімок)**: дозволяє зробити знімок екрана з каналу камери. Знімок зберігається в повній вихідній роздільній здатності потоку безпосередньо з кадрового буфера декодера, уникаючи будь-яких втрат через поточний розмір плитки вікна перегляду або масштабування роздільної здатності екрана. Успішне збереження знімка підтверджується блиманням жовтогарячого значка камери (`#ff7a00`) рівно 1 секунду.
2. **Піктограма відтворення (архів)**: використовується для швидкого відкриття архіву записів. Натискання цієї кнопки автоматично запускає шкалу часу `PlaybackWindow` для цієї конкретної камери, починаючи відтворення **рівно за 15 хвилин до поточного системного часу** (зручне швидке зміщення назад).
3. **Піктограма круглої стрілки (швидке відтворення/міні-програвач)**: активує швидке відтворення у вікні перегляду архівних записів камери за останні 30 хвилин безпосередньо в плитці камери, не виходячи з прямої сітки чи відкриваючи вікно повного архіву.
   * **Панель керування**: коли активна, у нижній частині вікна перегляду з’являється ковзаюча панель керування, яка містить:
     * **Відтворення/пауза**: призупиняє або відновлює відтворення потоку.
     * **Відображення часу та зміщення**: відображає позначку часу поточного відтворення та його відносний вік порівняно з реальним часом (наприклад, `-12:34` означає 12 хвилин і 34 секунди тому).
     * **Інтерактивний повзунок шкали часу**: дозволяє клацати або перетягувати для пошуку будь-де в межах 30-хвилинного буфера. Доступні сегменти запису виділено блакитним фоном, а червона точка відтворення позначає поточну позицію.
     * **Множник швидкості**: змінює швидкість відтворення (`1x`, `2x` та `4x`) для швидкого сканування відзнятого матеріалу.
     * **Закрити (✕)**: закриває міні-програвач і миттєво повертає вікно перегляду до живої стрічки.
   * **Підтримка Pannable Zoom**: так само, як у Live view, ви можете використовувати інструмент збільшувального скла, щоб збільшити область міні-програвача, або утримуйте середню кнопку миші та перетягуйте, щоб панорамувати.
4. **Піктограма 1:1 (власний масштаб)**: перемикає режим відображення відео піксель до пікселя. Якщо ввімкнено, відео не розтягується чи спотворюється, щоб заповнити межі плитки, а натомість центрується та відображається в оригінальній рідній роздільній здатності. Коли цей режим активний, фон кнопки та текст «1:1» підсвічуються яскравим неоновим світло-бірюзовим кольором.
5. **Піктограма збільшувального скла (інтерактивне масштабування)**: дозволяє збільшувати будь-яку певну область відеоканалу:
   * **Активація**: клацання значка перемикає його в активний стан (бірюзове виділення). Курсор змінює форму, а спливаюча підказка вказує: *«Натисніть і перетягніть подачу камери, щоб збільшити»*.
   * **Операція**: клацніть лівою кнопкою миші та перетягніть прямокутну область виділення на живу стрічку. Вікно перегляду автоматично обрізає та масштабує вибрану область, щоб заповнити всю плитку.
     * **Пропорційне масштабування**: утримуйте клавішу **Shift** під час малювання прямокутника масштабування, щоб змусити виділення зафіксувати співвідношення сторін 16:9, суворо обмежене межами вікна перегляду.
     * **Панорамування збільшеного зображення (Панорамне масштабування)**: після збільшення масштабу ви можете, утримуючи **середню кнопку миші (колесо прокручування)**, перетягувати курсор, щоб вільно панорамувати збільшене зображення навколо поля огляду відео. Це працює в Live view, архіві відтворення та міні-програвачі.
   * **Скинути**: під час масштабування піктограма лупи змінює свій значок (червона рамка зі знаком мінус). Натискання на неї негайно скидає масштаб, повертаючись до повної подачі камери.

## 5. Макети екрана, попередні налаштування та панель інструментів

Макети дозволяють організувати розташування камер на екрані. На вкладці **Попередні налаштування** (значок зірочки) ви можете:
* **Створіть нові стилі**: додайте власний макет із будь-якою конфігурацією стовпців і рядків (наприклад, 2x2, 3x3, 4x4).
* **Призначити камери**: клацніть вікно перегляду в макеті сітки, щоб вибрати його, потім відкрийте вікно камер NVR і натисніть кнопку **"+" (Додати)** на плитці потрібної камери. Ви також можете поміняти місцями вікна перегляду за допомогою контекстного меню (*„Zamień miejscami”* / *„Поміняти вікна перегляду”*) вихідного вікна перегляду, а потім клацнувши на цільовому вікні перегляду.

### Кнопки верхньої панелі (верхня панель інструментів)
Верхня розсувна панель інструментів містить повний набір кнопок навігації та керування програмами:
1. **Закрити вікно (червона кнопка ✕)**: закриває активне вікно. Щоб запобігти випадковим натисканням, він перехоплює подію закриття та пропонує діалогове вікно для підтвердження виходу з програми.
2. **Кнопка «Закріпити»**: керує поведінкою автоматичного приховання верхньої панелі інструментів. Коли шпилька спрямована вертикально (закріплений стан), смуга зафіксована на місці та залишається видимою назавжди. Коли штифт повертається на -45 градусів (у незакріпленому стані), смуга автоматично ковзає вгору з поля зору, коли миша залишає її область.
3. **Повноекранний режим (піктограма із зеленими стрілками)**: миттєво перемикає активне вікно в повноекранний режим і назад. У повноекранному режимі стрілки спрямовані всередину (згортання), а у віконному режимі – назовні (розгортання).
4. **Згорнути (піктограма мінімізації блакитного кольору)**: згортає вікно програми на панель завдань. Його відновлення повертає вікно до попереднього стану (наприклад, розгорнуте або повноекранний).
5. **⚙️ ПАРАМЕТРИ (OPCJE)**: перемикає вікно параметрів конфігурації. Якщо вікно вже відкрито, натискання цієї кнопки закриває його.
6. **📺 НОВЕ ВІКНО (NOWE OKNO)**: відкриває новий, незалежний і повністю конфігурований `Auxiliary Window`, ідеальний для розширення макетів вашої камери на кількох моніторах.
7. **АРХІВ**: відкриває порожній `PlaybackWindow` (програвач записів) з активною шкалою часу та календарем, що дозволяє вручну вибирати потік і канал камери з будь-якого налаштованого NVR через список бічної панелі.
8. **ІНСТРУКЦІЇ (INSTRUKCJA)**: відкриває це вікно посібника, завантажуючи повну документацію користувача англійською або польською мовами залежно від вашої активної мови.
9. **📊 Перемикач СТАТИСТИКА (STATYSTYKI)**: перемикач, щоб витягнути панель моніторингу системної статистики з лівого краю екрана.
10. **Перемикач блокування сітки (замок)**: коли перемикач увімкнено (виділено яскраво-помаранчевим), вимикає дії зі змінення розміру сітки на суміжних кнопках сітки, захищаючи макет активної камери від випадкових змін.
11. **Селектори розміру сітки (від 1x1 до 9x9)**: рядок із дев’яти кнопок, які дозволяють миттєво визначати структуру рядків і стовпців вашого вікна перегляду (від однієї камери 1x1 до 81 одночасної подачі камери в макеті 9x9). Поточний активний розмір підсвічується яскраво-помаранчевим кольором.
12. **Додаткові параметри (Гамбургер-меню з трьома рядками)**: кнопка, що відкриває пересувну панель інструментів `Layout & Grid Tools` для розширеного налаштування сітки, коригування геометрії та параметрів налагодження (докладно описано нижче).
13. **Кнопки попереднього налаштування/перегляду**: динамічно відтворені кнопки на правій стороні панелі інструментів, що представляють ваші налаштовані та видимі попередньо встановлені макети (наприклад, *📹 NVR*, *View 1* тощо). Натискання кнопки негайно перемикає сітку. Яскраві світло-бірюзові підсвічування Active View.

### Розширене налаштування сітки та співвідношення (інструменти макета та сітки)
Відкриття меню «Гамбургер» (Додаткові параметри) відкриває спеціальну панель інструментів макета. Щоб активувати елементи керування:
1. **Розблокувати панель інструментів**: перемкніть перемикач «Розблокувати панель інструментів» у самому верху. Це явний захід безпеки для запобігання випадковим змінам складних макетів.
2. **Користувацький розподіл вікон (F2 або натисніть і утримуйте)**: панель інструментів відображає кнопки розподілу сітки від 1x1 до 9x9. Надзвичайно просунутою функцією є можливість **перевизначати та редагувати розміри розділів**. Якщо клацнути й утримувати будь-яку кнопку сітки лівою кнопкою миші (або сфокусувати її та натиснути **F2**), з’явиться текстове поле. Ви можете ввести будь-який нестандартний або асиметричний поділ (наприклад, `2x3`, `1x4` тощо) і натиснути Enter. Кнопка миттєво перепрограмується, і натискання її застосовує ваш спеціальний макет до головного вікна перегляду.
3. **Геометричні співвідношення**: дозволяє примусово відображати сітку до певних пропорцій:
   * **Співвідношення сторін 16:9**: блокує та масштабує контейнер сітки до широкоформатного формату 16:9 (стандарт для сучасних IP-камер).
   * **Співвідношення сторін 4:3**: адаптує контейнер сітки до традиційного співвідношення сторін 4:3 (поширене у застарілих аналогових/IP-камерах).
4. **Операції з сіткою (об’єднати виділені клітинки)**: доступ до функції асиметричного об’єднання клітинок (докладно в розділі 9.2).

## 6. Панель системної статистики (Системна статистика)

Ця панель, що висувається з лівого краю екрана Live View, відстежує стан комп’ютера та навантаження, створюване програмою:
* **Параметри, що контролюються**:
  * **ЦП / ОЗП**: використання основного процесора (у % від усіх ядер) і оперативної пам’яті, що безпосередньо використовується процесом `kvision` і пов’язаними з ним підпроцесами завантажувача.
  * **GPU / VRAM**: використання ядра відеокарти (у %) і обсяг графічної пам’яті VRAM, зайнятої рендерингом і апаратним декодуванням (підтримує повний список процесів GPU за допомогою аналізатора XML із `nvidia-smi`).
  * **МЕРЕЖА (Мережа)**: Фактична швидкість передачі завантажень програми з усіх активних живих програвачів, архівних програвачів і процесів завантаження записів.
* **Багатопотоковість (без заїдань)**: збір даних процесу та GPU виконується в окремому системному потоці (`StatsWorker`). Це запобігає будь-якому мікрозаїданню під час візуалізації відео (без падінь кадрів).
* **Функція закріплення**: натискання кнопки **"Закріпити"** (піктограма шпильки) блокує панель у розгорнутому стані.
* **Естетика**: діаграми мають яскраві неоново-зелені межі, градієнтну заливку під кривою діаграми та збалансовану прозорість фону на 35%, щоб забезпечити читабельність тексту.

## 7. Програвач архівів відтворення

Доступно, натиснувши піктограму годинника/відтворення поруч із певною камерою чи рекордером. Це дозволяє одночасно переглядати архівні записи з кількох камер Hikvision у повній синхронізації.

### Хронологія та елементи керування:
* **Швидкий старт (15 хвилин назад)**: коли відкриваєте архів у режимі реального часу, програвач автоматично запускається з моменту, який **рівно на 15 хвилин раніше поточного системного часу** (замість початку опівночі). Це дозволяє негайно переглянути подію, яка щойно сталася.
* **Навігація**: шкалу часу можна прокручувати ліворуч і праворуч, перетягуючи її лівою кнопкою миші.
* **Масштаб (масштабування)**: ви можете плавно змінювати масштаб шкали часу за допомогою коліщатка прокручування миші (або кнопок масштабування) – від перегляду всього дня до точного перегляду з точністю до 10 хвилин.
* **Ярлики швидкого масштабування**: на нижній панелі керування є спеціальні кнопки з круглими значками для миттєвого масштабування перегляду часової шкали:
  * **Піктограма „1h”**: збільшує часову шкалу для детального огляду протягом 1 години.
  * **Піктограма „8h”**: масштабує часову шкалу для відображення 8-годинного періоду.
  * **Піктограма „24h”**: скидає масштаб для розміщення повної 24-годинної доби на одному екрані.
  * **Піктограма «Центр»**: миттєво центрує часову шкалу так, щоб червоний індикатор відтворення знаходився точно посередині екрана (замінює стару текстову кнопку).
* **Навігація за датою (календар і дні)**: елементи керування поруч із відображеною датою дозволяють швидко переходити:
  * Кнопки **„<” (Попередній день)** і **„>” (Наступний день)**: дозволяють миттєво перейти на 24 години назад або вперед, не відкриваючи діалогового вікна календаря.
  * **Піктограма календаря**: відкриває спливаюче вікно календаря для вибору певної дати.
  * **Піктограма оновлення**: примусовий новий пошук записів. Натискання на неї очищає локальний кеш сегментів доступності запису для всіх активних каналів і надсилає нові запити до NVR, що дуже корисно для завантаження файлів, записаних кілька хвилин тому (замінюючи стару текстову кнопку).
  * **Сьогодні (піктограма сьогоднішньої дати/числа)**: миттєво повертається до поточного дня (замінює стару текстову кнопку).
* **Ярлики швидкості відтворення**:
  * **Значки „1x”, „2x” і „4x”**: Миттєво змініть множник швидкості відтворення.
* **Кнопки переходу відеомагнітофона**:
  * **Піктограми „15”, „45” та „60” з круглими стрілками**: дозволяють швидко перейти назад або вперед на вказану кількість секунд.
* **Смуги доступності запису**: Кольорові смуги, що представляють знайдені сегменти відео на диску записуючого пристрою, відображаються під шкалою часу. Система кешування запобігає мерехтінню під час перетягування.
* **Автовідстеження (відстеження індикатора)**: індикатор відтворення (вертикальна червона лінія) постійно контролюється. Якщо індикатор виходить за межі видимого діапазону шкали часу, подання автоматично прокручується до центру. Ця опція інтелектуально блокується під час ручного перетягування індикатора користувачем.

### Бічна панель камери у вікні відтворення
У вертикальному бічному списку на правому краю програвача перераховані всі налаштовані NVR та їхні канали камер:
* **Перемикання каналів**: клацання будь-якого каналу камери у списку додає його як активний слот для відтворення на часовій шкалі (створюється відеоплеєр). Повторне клацання канал видаляє.
* **Контекстне меню каналу**: клацання правою кнопкою миші активного відеослоту в сітці відтворення відкриває меню для:
  * Перемикання якості відео (основний потік / допоміжний потік).
  * Закрити/видалити активний плеєр із відтворення архіву.

## 8. Завантаження записів (Завантажувач)

У вікні «Архів відтворення» ви можете завантажити вибрані сегменти записів безпосередньо на диск комп’ютера як файли MP4:
1. Натисніть значок завантаження (стрілка вниз) поруч із вибраною камерою.
2. Виберіть діапазон часу (початок і кінець запису).
3. Виберіть місце збереження файлу призначення.
4. Натисніть **Завантажити**.

### Розширені функції завантаження:
* **Послідовне завантаження сегментів (частини по 1 ГБ)**: програма автоматично розбиває ваш запит часового діапазону на фізичні сегменти файлів (приблизно 1 ГБ кожен на диску NVR) і завантажує та конвертує їх по одному (використовуючи тимчасові файли `.pspart`, які перетворюються безпосередньо у формат `.mp4`). Це забезпечує високостабільні завантаження тривалого діапазону без переповнення пам’яті або зависань перетворення FFmpeg.
* **Візуалізація загального прогресу**: панель прогресу (яскраво-блакитного кольору) відображає загальний прогрес завантаження для камери в усіх сегментах. Текст стану, накладений на панель виконання, показує поточну частину та відсотки, наприклад, `Downloading part 1 of 3... 45% (Overall: 15%)`, із контурним стилем, щоб гарантувати розбірливість на будь-якому тлі.
* **Очищення IP-адрес файлів**: назви відеофайлів (і миттєві/архівні знімки) автоматично позбавляються IP-адрес NVR/DVR, щоб вони були чистими та зрозумілими для людини (наприклад, `4_Wejscie_glowne_2026-06-15.mp4` замість `<RECORDER_IP>_4_Wejscie...`).

## 9. Розширені налаштування та налаштування у вікні параметрів

Вікно параметрів і конфігурації (`SideBar`) складається з шести спеціальних вкладок:

### 1. Деталі вікна перегляду (Піктограма монітора)
Відображає розширені параметри поточної вибраної плитки сітки. Дозволяє:
* Введіть спеціальну **URL-адресу основного потоку** (RTSP/ONVIF) і **URL-адресу додаткової резервної копії** для налаштування вручну.
* Перемикання/вмикання звуку аудіоканалу вибраної камери.
* Введіть розширені перевизначення декодера в текстовому полі **Перевизначення параметрів FFmpeg**.
> [!ПОРАДА]
> Для найшвидшого потокового з’єднання та максимальної стабільності через RTSP рекомендовані такі параметри:
> ```ini
> -analyzeduration 0 -probesize 500000 -fflags nobuffer -flags low_delay -rtsp_transport tcp
> ```

### 2. Інструменти макета та сітки (Піктограма повзунків)
Розширені параметри налаштування сітки екрана:
* Швидкий перемикач для повноекранного режиму.
* **Асиметричне об’єднання комірок (об’єднати виділені комірки)**: надзвичайно просунутий редактор макета. Утримуйте **Ctrl** або **Shift** і клацніть, щоб вибрати кілька суміжних плиток у сітці, або скористайтеся клавіатурою, утримуючи **Shift** і переміщаючись за допомогою **клавіш зі стрілками**, а потім натисніть «Об’єднати виділені клітинки», щоб об’єднати їх в одне більше вікно перегляду. Це дає змогу створювати повністю індивідуальні асиметричні дизайни сітки (наприклад, одна величезна панель камери з меншими каналами, що оточують її).

### 3. Записувачі (Піктограма сервера)
Повний менеджер конфігурації для підключення до пристроїв Hikvision NVR/DVR (детально описано в розділі 3).

### 4. Попередні налаштування (Зірочка)
Менеджер для збережених макетів сітки та призначень камер. Дозволяє створювати порожні шаблони сіток, змінювати їх порядок, перемикати видимість їх верхньої панелі (за допомогою перемикача «Видимі») або активувати їх у поточному вікні.

### 5. Налаштування (значок шестірні)
Дозволяє налаштувати глобальні параметри програми:
* **Дозволити запуск кількох екземплярів**: позначка цього пункту дозволяє запускати кілька паралельних копій процесу KVision (за замовчуванням це обмежує запуски одним активним екземпляром).
* **Налаштування автоматичного згортання**: налаштовує час ковзання анімації для верхньої панелі та панелі статистики.
* **Дозволити зміну вікон перегляду**: прапорець *„Дозволити зміну місць вікна перегляду”* дозволяє змінювати положення камери на сітці на льоту (клацніть правою кнопкою миші вихідну плитку -> виберіть "Поміняти місцями перегляду" -> клацніть лівою кнопкою миші цільову плитку).
* **Дозволи для натискання правою кнопкою миші**: перемикання на блокування/розблокування інтерактивних елементів керування в контекстному меню, що відкривається правою кнопкою миші (Увімкнути контекстне меню, дозволити змінювати вікна перегляду, увімкнути параметр «Видалити камеру», дозволити змінювати налаштування вікна перегляду, увімкнути вибір якості основного/допоміжного потоку).
* **Автоматичне ввімкнення звуку**: автоматично вмикає аудіопотік активного вікна перегляду під час переходу в повноекранний режим.
* **Приховати курсор у повноекранному режимі**: прапорець *„Приховати курсор у повноекранному режимі”* автоматично приховує курсор миші після короткого періоду бездіяльності під час повноекранного перегляду, щоб забезпечити безперешкодний перегляд.
* **Вибір мови**: миттєво перемикає переклад інтерфейсу (за замовчуванням у системі, польська, англійська).
* **Параметри інтерфейсу користувача**: приховати/показати мітки стану вікна перегляду або значки керування (наприклад, автоматичне приховування накладень елементів керування в нижньому правому куті плиток, якщо не навести курсор).
* **Параметри FFmpeg за замовчуванням і масове оновлення**: дозволяє налаштовувати глобальні параметри FFmpeg, застосовані до нових вікон перегляду (рекомендовано: `-analyzeduration 100000 -probesize 500000 -fflags nobuffer -flags low_delay`).
  * > **[РЕКОМЕНДАЦІЯ]** Якщо ви переходите зі старішої версії KVision (до версії 2.4.5), наполегливо рекомендуємо змінити `-analyzeduration 0` на `-analyzeduration 100000` у глобальних налаштуваннях і натиснути «Оновити всі камери». Налаштування `0` для потоків без аудіодоріжки спричиняє затримки під час перемикання, оскільки FFmpeg блокуватиме 5-секундний час очікування аудіопакетів. Встановлення дуже низького значення (наприклад, 100000 мікросекунд = 0,1 с) повністю вирішує цю проблему, зберігаючи миттєве перемикання.
  * **Кнопка Оновити всі камери**: кнопка *„Zaktualizuj wszystkie kamery”* застосовує ці глобальні параметри до всіх налаштованих вікон перегляду.
  * **Позначки низької затримки (`nobuffer`, `low_delay`)**:
    * `-fflags nobuffer` вимикає внутрішню буферизацію пакетів демультиплексора, щоб усунути затримку потокового передавання з часом, але може викликати затримки, якщо ваше мережеве з’єднання з камерою нестабільне.
    * `-flags low_delay` вказує декодеру негайно виводити кадри, але може спричинити короткі артефакти/змазування в потоках, які використовують B-кадри під час швидких рухів.
  * **Перевизначати прапорець**: ви можете виключити певні вікна перегляду з глобальних оновлень або динамічних параметрів за замовчуванням, позначивши **"Nie uwzględniaj zmian w globalnych ustawieniach FFMpeg"** у окремому діалоговому вікні налаштувань вікна перегляду. Корисно для усунення несправностей камер, що зависають.


### 6. Журнал змін (значок годинника/документа)
Представляє інтерактивну хронологію, яка показує повну історію випусків, оновлення, виправлення помилок і додавання функцій KVision, що гарантує вам прямий доступ до деталей оновлень програми.

* **Колесо прокрутки миші**:
  * Налаштовує масштаб (масштаб) шкали часу в програвачі архівів відтворення.

## 11. Створення знімків і налаштування шляху

Програма дозволяє швидко робити високоякісні знімки з будь-якого вікна огляду камери як у режимі Live View, так і в режимі Playback Archive.

### Зйомка миттєвих знімків:
1. Кнопка накладання піктограми камери доступна в нижньому правому куті кожного вікна перегляду (докладніше в розділі 4).
2. Натискання піктограми камери знімає кадр і зберігає його як зображення JPEG (якість 98 – практично без втрат).
3. Успішне захоплення підтверджується тим, що піктограма камери блимає оранжевим (`#ff7a00`) рівно 1 секунду.
4. **Повна роздільна здатність**: у режимі архіву відтворення знімки зберігаються з повною вихідною роздільною здатністю потоку безпосередньо з кадрового буфера декодера, незалежно від розміру активного вікна перегляду на екрані чи масштабування дисплея.

### Збереження конфігурацій шляху:
1. Перейдіть на вкладку **Налаштування** (значок шестірні у вікні параметрів).
2. У розділі **Збереження** ("Запис") ви можете налаштувати шляхи за замовчуванням:
   * **Шлях до знімків за умовчанням**: папка, у якій зберігатимуться знімки (за замовчуванням `~/Obrazy/CCTV`).
   * **Шлях до записів за замовчуванням**: папка, у якій зберігатимуться завантажені відео MP4 (за замовчуванням `~/Wideo/CCTV`).
3. Натискання кнопки браузера `...` відкриває вихідний селектор папок вашої операційної системи (Breeze у KDE).
4. **Поведінка кнопки браузера**: засіб вибору каталогу відкривається саме за шляхом, введеним у текстове поле (якщо воно існує). Якщо поле порожнє, недійсне або вказує на папку, до якої ви не маєте доступу, діалогове вікно повертається назад і відкривається у вашому домашньому каталозі (`~/`).

### Налаштування інтерфейсу користувача (UI):
1. Перейдіть на вкладку **Налаштування** (піктограма шестірні у вікні параметрів) або відкрийте вікно **Параметри** ("Opcje").
2. У розділі **Налаштування інтерфейсу користувача** ви можете налаштувати видимість елементів, які накладаються на камеру kafelki/viewports:
   * **Показувати статус каналу у верхньому лівому куті вікна перегляду** (за замовчуванням увімкнено) — відображає інформацію про завантаження потоку, відтворення та статус з’єднання.
   * **Показувати інформацію про камеру в нижньому лівому куті вікна перегляду** (за замовчуванням увімкнено) — відображає назву камери, отриману з рекордера Hikvision.
   * **Показувати піктограми керування в нижньому правому куті вікна перегляду лише під час наведення курсора** (за замовчуванням увімкнено) — автоматично приховує панель кнопок керування (знімок, архів, 1:1 піксель до пікселя, масштабування області), коли курсор миші знаходиться за межами певного вікна перегляду камери. Піктограми з’являються миттєво, щойно ви наводите курсор миші на вікно перегляду (не потрібно клацати), і зникають, коли виходите з нього, максимізуючи видимість потоків камери.
   * **Показувати інформаційні поля лише під час наведення курсора** (за замовчуванням вимкнено) — аналогічний параметр, який приховує статус у верхньому лівому куті та ім’я у нижньому лівому куті вікон перегляду, відображаючи абсолютно чистий потік камери, якщо курсор не переміщується над цією конкретною плиткою камери.
